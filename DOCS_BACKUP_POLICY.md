# バックアップ運用ポリシー（ローカル管理へ移行）

目的: Git リポジトリ上でバックアップブランチやバックアップコミットを作成する運用をやめ、ローカルでのバックアップ管理へ切り替えます。

要点:

- バックアップは `local_backups/` 配下にタイムスタンプ付きアーカイブとして保存します。リポジトリには追加しません。\
- 日常的なバックアップ作業には `scripts/local_backup.sh` を利用してください。\
- リモートに `backup/...` ブランチを作成して push しないでください。誤って push しないよう、`scripts/install_git_hooks.sh` で pre-push フックの導入を推奨します。\
- 既にリモートにあるバックアップブランチが不要なら削除します（運用チームで合意の上で実施）。\

運用例:

1. ローカルでバックアップを作る:

```
./scripts/local_backup.sh "docs learning_materials"
```

2. フックをインストールする（初回のみ）:

```
./scripts/install_git_hooks.sh
```

注意: 履歴に大きなファイル（例: `corpus/bench_bge.index`）が残っていると、`main` への push が失敗します。履歴の書換えを行う場合はチーム合意とバックアップを事前に取ってください。
