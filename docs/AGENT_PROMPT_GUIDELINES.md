# エージェント応答ガイドライン

目的: ユーザとの会話で文脈ずれや無関係な追加質問を減らすための簡易プロンプトテンプレートと運用ルール。

1) 応答テンプレート（システム / アシスタント用）

- ステップ1 — ユーザ発話の復唱（1文）: 「理解確認: <簡潔な要約>」
- ステップ2 — 必要な追加情報の明示的確認（該当する場合）: 「確認したい点: <具体的な1-2項目>」
- ステップ3 — 回答本体（要点を先に）：短い結論→必要なら詳細

例:

理解確認: あなたはトウモロコシ30本の植栽に必要なスペースを知りたい、という認識で合っていますか？

確認したい点: 1) 畝幅の想定はありますか？ 2) 防風対策は必要ですか？

回答: 一般的には〜（以下省略）

2) 運用ルール
- 直接答えられる質問はまず短く答え、続けて「念のため確認」項目を提示する。
- 追加質問は最大2項目までに制限し、関連性を明記する。
- ユーザの最終発話が明確なら復唱は省略可能だが、曖昧な場合は必ず復唱する。

3) 実装案
- すぐ反映できるのはエージェント側の「応答スタイル」変更（プロンプト設計）。
- サーバ／コードレベルでの変更が可能なら、レスポンス前にユーザ発話をパースして要約を自動挿入するラッパーを追加する。

4) テスト
- 会話例を数件用意して自動検証する（期待される復唱・確認が含まれているか）。
