## 🎯 本ガイドについて

このガイドは、Phase 7 + RAG Agent 統合システムを実際に使用する際に必要な情報をまとめています。

### 対象ユーザー

- **医療専門家**: 医学的情報の検索・統合
- **法律専門家**: 法律情報の高度な検索・分析
- **技術管理者**: システム管理・監視
- **ビジネス管理者**: 事業戦略・実装計画

## 🚀 システム基本操作

### 1. システムへのアクセス

```bash
# Web UI によるアクセス
http://localhost:8501

# または Docker コンテナ内で実行
docker-compose up
```

### 2. 基本的なクエリ

システムは複数のドメイン知識を統合して回答を生成します。

**例1: 医療クエリ**
```
Q: "新しい治療法の臨床効果と既存治療との比較"

期待される動作:
✓ Primary Domain: Medical (医学)
✓ Related Domains: Science (科学), Legal (法律)
✓ 統合回答: 医学的知見 + 科学的根拠 + 法的側面
```

**例2: 法律クエリ**
```
Q: "契約交渉における技術条項の解釈"

期待される動作:
✓ Primary Domain: Legal (法律)
✓ Related Domains: Technical (技術)
✓ 統合回答: 法的解釈 + 技術的背景
```

### 3. ドメイン別利用方法

#### 医療分野 (Medical)

**使用ケース**:
- 治療法・医学用語の検索
- 臨床判断の裏付け情報
- 医療倫理に関する相談

**おすすめクエリ形式**:
```
"[症状/治療] の [科学的根拠/比較/予後評価]"

**実装例**:
```
- "がん治療における免疫療法と従来療法の比較"

- "公開鍵基盤を使用した医療データ保護"

#### 法律分野 (Legal)

**おすすめクエリ形式**:

**実装例**:

## 🎤 音声入力機能

### 概要

マイクから録音した音声を自動でテキストに変換し、そのままクエリとして送信できます。  
音声認識には **faster-whisper**（Whisperモデル）を使用し、日本語に最適化されています。

### 使い方

1. メイン画面の **「🎤 音声入力」** エクスプローラーを展開する
2. マイクアイコンをクリックして録音を開始する
3. 話し終えたら録音を停止する
4. 自動で文字起こしが実行され、テキストが表示される
5. 必要に応じてテキストエリアで内容を編集する
6. **「🚀 このテキストを送信」** ボタンを押してLLMに送信する

> **クリアボタン**: 「🗑️ クリア」で文字起こし結果を破棄できます。

### 音声認識モデルの選択

サイドバーの **「🎨 マルチモーダル設定 → 音声認識」** から使用するモデルを選択できます。

| モデル | 速度 | 精度 |
|--------|------|------|
| whisper-tiny | 最速 | 標準 |
| whisper-small | 中程度 | 高い |
| whisper-base | やや遅い | より高い |

> 通常の利用には `whisper-tiny` を推奨します。精度を重視する場合は `whisper-small` 以上を選択してください。

### 注意事項

- ブラウザからマイクへのアクセス許可が必要です
- 録音は `.wav` 形式で処理されます
- 文字起こし中は「🔄 音声を文字起こし中...」のスピナーが表示されます
- `faster-whisper` がインストールされていない場合は音声入力セクションが非表示になります

## ⚙️ 高度な機能

### 1. マルチドメイン統合検索

複数のドメインにまたがるクエリに対して、自動的に関連ドメイン情報を統合します。

```
Q: "医療技術導入のビジネス的実現可能性"

自動処理:
1. Primary: Business ドメイン認識
2. Related: Medical, Technical ドメイン検出
3. Integration: 3つのドメイン知識を統合
4. Response: 医療 + 技術 + ビジネスの観点から回答
```

### 2. 信頼度スコア

システムは各回答に対して信頼度スコアを提供します。

```
Response: "..."
Confidence: 95%  (医学的根拠が十分)
Related Domains: Science (98%), Legal (75%)
```

### 3. 関連情報の提示

メイン回答に加えて、関連する他のドメイン情報も提示されます。

## 🔍 トラブルシューティング

### 問題1: ドメイン推定が不正確

**症状**: 期待したドメインが正しく認識されない

**対応**:
1. クエリをより具体的に記述
2. ドメインキーワードを明示的に含める
3. 複数のクエリバリエーションで試す

**例**:
```
❌ 不正確: "新しい方法について"
✅ 正確: "医療分野における新しい治療法の効果"
```

### 問題2: レスポンスが遅い

**症状**: 回答生成に時間がかかる

**対応**:
1. ネットワーク接続を確認
2. システムリソースを監視 (CPU/メモリ)
3. 複雑なクエリを分割して実行

### 問題3: 特定のドメイン情報が不十分

**症状**: 関連ドメイン情報が提示されない

**対応**:
1. ナレッジベースの更新を確認
2. システム管理者に連絡
3. 代替クエリ表現を試す

### 問題4: 音声入力が表示されない

**症状**: 「🎤 音声入力」セクションが画面に表示されない

**原因**: `faster-whisper` パッケージが未インストール

**対応**:
```bash
pip install faster-whisper
```
インストール後、ページを再読み込みしてください。

### 問題5: 音声の文字起こし精度が低い

**症状**: 話した内容と異なるテキストが出力される

**対応**:
1. サイドバーで `whisper-small` または `whisper-base` に変更する
2. 静かな環境で録音する
3. マイクに近づいてはっきり話す
4. 文字起こし後のテキストエリアで手動修正する

---

### クエリ作成のコツ

1. **具体性**: 一般的ではなく、具体的な内容を記述
2. **複合性**: 複数ドメインにまたがるクエリは新しく記述
3. **背景説明**: 文脈情報を含める

### 回答の活用

1. **複数試行**: 異なる言い回しで試す
2. **信頼度確認**: 信頼度スコアを参考にする
3. **関連情報活用**: 関連ドメイン情報も検討
