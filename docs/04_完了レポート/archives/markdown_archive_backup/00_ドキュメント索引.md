## 🎯 ドキュメント体系の全体像

このプロジェクトのドキュメントは、ユーザーの段階と用途に応じて **8つのカテゴリ** に体系化されています。

```
┌─────────────────────────────────────────────────────────────┐
│                    ドキュメント体系                         │
├─────────────────────────────────────────────────────────────┤
│  📖 基礎段階                                                │
│  ├─ 01_クイックスタート        ← まずここから！           │
│  └─ 02_ユーザーガイド                                      │
│                                                             │
│  🔧 実行段階                                                │
│  ├─ 03_運用ガイド              ← 本番運用者向け           │
│  └─ 06_テスト・検証                                        │
│                                                             │
│  🏗️ 開発・技術段階                                          │
│  ├─ 04_技術ドキュメント        ← エンジニア向け           │
│  └─ 05_本番環境               ← インフラ向け           │
│                                                             │
│  📚 学習・参考段階                                          │
│  ├─ 07_学習資料               ← 深掘り学習向け           │
│  └─ 08_チェンジログ・レポート ← 過去記録・参考資料       │
└─────────────────────────────────────────────────────────────┘
```

## 📑 各カテゴリの詳細

### 🚀 01_クイックスタート
**対象**: 初めてシステムを使う方  
**目的**: 最短で基本操作を習得

| ファイル | 説明 | 読むべき人 |
|---------|------|----------|
| クイックスタート_初心者向け.md | 5分で基本操作を習得 | 全員 |
| システム概要.md | システムの全体構造 | 全員 |
| AIベギナーガイド.md | AI・RAG技術の基礎 | 初心者向け |

**📍 このカテゴリを読む時間**: 15～30分

---

### 👥 02_ユーザーガイド
**対象**: 日常的にシステムを使用する方  
**目的**: 目的別に機能を活用する

| ファイル | 説明 | 使用場面 |
|---------|------|--------|
| ユーザー操作ガイド.md | 基本操作・設定方法 | 日常業務 |
| 利用事例とFAQ.md | 実際の使用例・よくある質問 | トラブル時 |
| ドメイン別機能説明.md | 医療・法律・技術など各分野の使い方 | 専門分野での利用 |
| トラブルシューティング.md | 問題発生時の対処 | エラー時 |

**📍 このカテゴリを読む時間**: 必要に応じて各1～5分

---

### 🔧 03_運用ガイド
**対象**: システム運用・管理担当者  
**目的**: 本番環境を安定稼働させる

| ファイル | 説明 | 実行頻度 |
|---------|------|--------|
| 運用開始ガイド.md | サーバー起動・初期設定 | 初回のみ |
| 本番環境監視ガイド.md | リソース監視・アラート対応 | 毎日 |
| デプロイメントガイド.md | アップデート・新機能展開 | 月1～2回 |
| ロールバック手順.md | 問題時の復旧手順 | 必要時 |

**📍 このカテゴリを読む時間**: 30分～2時間(初回)、以降は参照用

---

| LLM推論エンジン説明.md | AIエンジンの仕組み | ML技術者 |

**📍 このカテゴリを読む時間**: 1～3時間(初回)、参照用

---

**対象**: インフラストラクチャ管理者  
**目的**: 本番環境の状態・検証結果を確認

| ファイル | 説明 | 確認時期 |
|---------|------|--------|
| デプロイメント完了報告.md | 本番デプロイ実施記録 | デプロイ後 |
| デプロイメント検証完了.md | デプロイ検証結果 | デプロイ後 |
| 本番テスト完了.md | 本番環境の最終検証 | デプロイ後 |

**📍 このカテゴリを読む時間**: 必要に応じて各15～30分

---

### ✅ 06_テスト・検証
**対象**: QA・テスト担当者  
**目的**: テスト計画・結果を確認・実行

| ファイル | 説明 | タイミング |
|---------|------|----------|

| 本番テスト結果.md | テスト実行結果 | デプロイ後 |

**対象**: システム・技術の深い理解を求める方

**📍 このカテゴリを読む時間**: 自習ペースに応じて (数学基礎: 5～10時間、ニューラルネット: 8～15時間)

## 🔄 ドキュメント拡張ポリシー

このプロジェクトは継続的に進化します。新しいドキュメントを追加する際のガイドライン：

### 📌 新規ドキュメント追加時
1. **カテゴリ決定**: 上記8つのカテゴリのいずれかに必ず含める
2. **ファイル名**: `説明的な日本語名+.md` 形式
3. **インデックス更新**: このファイルのチェックリストを更新
4. **内容**: 対象ユーザーを明記し、読了所要時間目安を記載

### 🏷️ ファイル名の命名規則
- ✅ **良い例**: `ユーザー操作ガイド.md`, `本番環境監視ガイド.md`
- ❌ **悪い例**: `USER_GUIDE.md`, `guide.md`, `新しいドキュメント.md`
- **規則**: 「名詞+説明的修飾語+.md」形式

### 📂 フォルダ構成の保守
```
docs/
├── 00_ドキュメント索引.md          ← マスターインデックス（このファイル）
├── 01_クイックスタート/            ← 初心者向け 3～5ファイル
├── 02_ユーザーガイド/              ← ユーザー向け 4～6ファイル
├── 03_運用ガイド/                 ← 運用者向け 4～6ファイル
├── 04_技術ドキュメント/            ← 開発者向け 5～8ファイル
├── 05_本番環境/                   ← インフラ向け 3～5ファイル
├── 06_テスト・検証/                ← QA向け 3～5ファイル
├── 07_学習資料/                   ← 学習向け 3～10ファイル
└── 08_チェンジログ・レポート/      ← 管理向け 4～10ファイル
```
