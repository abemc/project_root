## STEP 1️⃣: ドメイン分類をテスト（15分）

### テスト1: ドメイン分類システムの確認

このコマンドをコピー&ペーストして実行：

```bash
cd /home/abemc/project_root

python3 << 'EOF'
import sys
sys.path.insert(0, 'src')

from self_improvement.domain_knowledge import DomainKnowledgeManager

print("【ドメイン分類システムのテスト】\n")

# ドメイン管理システムを初期化
dm = DomainKnowledgeManager()

# テストクエリ
queries = [
    "患者の治療方法について教えてください",
    ("知的財産権の登録手続きについて", "legal"),
    "デジタルトランスフォーメーション戦略",
    "クラウドインフラの構築方法",
    "機械学習モデルの精度評価",
]

print("テスト開始...\n")

for query in queries:
    # クエリが複数値の場合と単一値の場合を処理
    if isinstance(query, tuple):
        q, expected = query
    else:
        q = query
        expected = None
    
    domain, confidence = dm.infer_domain_from_query(q)
    
    print(f"Q: {q}")
    print(f"   推定ドメイン: {domain}")
    print(f"   信頼度: {confidence:.1%}")
    if expected:
        print(f"   期待値: {expected}")
    print()

print("✅ ドメイン分類テスト完了")
EOF
```

**期待される出力:**
```
Q: 患者の治療方法について教えてください
   推定ドメイン: medical
   信頼度: 95%

Q: 知的財産権の登録手続きについて
   推定ドメイン: legal
   信頼度: 88%

...
```

✅ **成功チェック**: すべてのクエリが正しくドメイン分類されたか？

## 🎮 補足：失敗した場合

### エラー: モジュールが見つからない

```
ModuleNotFoundError: No module named 'src.xxx'
```

**解決方法:**
```bash
# パスが正しくセットされているか確認
export PYTHONPATH=/home/abemc/project_root:$PYTHONPATH

# 再度実行
python3 your_script.py
```

### エラー: 環境問題

```
python --version
# Python 3.8以下の場合は3.9以上にアップグレード必要

# 仮想環境を確認
which python
# /home/abemc/project_root/.venv/bin/python なら OK
```

### テストが失敗

```
# 詳細なエラーを確認
python -m pytest tests/test_phase7_integration.py -vv

# 1つのテストだけを実行
python -m pytest tests/test_phase7_integration.py::test_domain_inference -v
```
