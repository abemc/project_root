## 1️⃣ Level 1: 基本概念

### 1.1 ベクトル (Vector)

**定義**: データポイントを多次元空間で表現したもの

```python
# 例: 映画の特性をベクトルで表現
映画A = [面白さ: 8.5, アクション: 7, コメディ: 3]  # 3次元ベクトル

# 数学的表記
v = [v₁, v₂, v₃] = [8.5, 7, 3]
```

**プロジェクトでの使用**:
- クエリを数値ベクトルに変換
- ドキュメントを数値ベクトルに変換
- ベクトル空間で「意味の近さ」を計算

### 1.2 ベクトルの大きさ（ノルム）

**ノルム (L2 ノルム)**: ベクトルの「長さ」

$$\lVert \mathbf{v} \rVert = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2}$$

```python
# 例
v = [3, 4]
||v|| = √(3² + 4²) = √(9 + 16) = √25 = 5
```

**プロジェクトでの使用**: 異なるノルムを持つベクトルを正規化

### 1.3 ベクトルの内積（ドット積）

**内積**: 2つのベクトルの「方向の一致度」を示す

$$\mathbf{u} \cdot \mathbf{v} = u_1v_1 + u_2v_2 + \cdots + u_nv_n$$

```python
# 例
u = [1, 2]
v = [3, 4]
u · v = 1×3 + 2×4 = 3 + 8 = 11
```

**プロジェクトでの使用**: 類似度計算の基礎

### 1.4 行列 (Matrix)

**定義**: 数字を行と列で整理したもの

```
行列 M = [
  [1, 2, 3],     ← 1行目
  [4, 5, 6],     ← 2行目
  [7, 8, 9]      ← 3行目
]
```

**プロジェクトでの使用**:
- 複数のドキュメント埋め込みを同時処理
- トランスフォーマーの重み行列
- キャッシュデータ構造

### 1.5 確率の基本

**確率**: ある事象が起きる可能性（0～1）

```
P(事象) = 起きる場合の数 / 全ての場合の数

例: サイコロで「3」が出る確率
P(3) = 1/6 ≈ 0.167
```

**プロジェクトでの使用**:
- LLM の生成確率
- 各単語が出現する確率
- ドメイン推定の確信度

### 1.6 距離・類似度の計算

#### 📏 ユークリッド距離

$$d = \sqrt{(x_1-x_2)^2 + (y_1-y_2)^2}$$

```python
# 例: 2次元空間での距離
p1 = [0, 0]
p2 = [3, 4]
distance = √((3-0)² + (4-0)²) = √(9+16) = 5
```

#### 📊 コサイン類似度 ⭐ **重要**

$$\text{similarity} = \frac{\mathbf{u} \cdot \mathbf{v}}{\lVert \mathbf{u} \rVert \times \lVert \mathbf{v} \rVert}$$

- **プロジェクトで最も使用される計算**
- 値: -1 ～ 1（1に近いほど類似）
- ベクトルの大きさに左右されない（方向のみを比較）

```python
# 実装例: RAG 検索で使用
query = "新しい治療法"  # をベクトル化 → [0.2, 0.8, 0.1, ...]
doc1 = "最新の治療方法"  # をベクトル化 → [0.19, 0.81, 0.12, ...]
doc2 = "猫の飼い方"      # をベクトル化 → [0.01, 0.02, 0.03, ...]

similarity(query, doc1) ≈ 0.99  ← 似ている
similarity(query, doc2) ≈ 0.05  ← 似ていない
```

## 2️⃣ Level 2: 応用概念

### 2.1 埋め込み (Embedding)

**定義**: テキストを数値ベクトルに変換する処理

```
"新しい治療法について"
    ↓
[0.123, -0.456, 0.789, 0.234, -0.567, ...]  ← 384次元ベクトル
    ↓
意味情報を保持した数値表現
```

**特性**:
- 意味が近い文は、ベクトル空間でも近い位置
- ベクトルの距離 ∝ 文の意味的距離

**プロジェクトでの実装**:
```python
# src/embeddings/ で使用
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('sentence-transformers/...')
embeddings = model.encode(texts)
# 出力: [[0.123, -0.456, ...], [0.234, -0.567, ...], ...]
```

### 2.2 RAG (Retrieval-Augmented Generation)

**プロセス**:

```
1️⃣ 検索フェーズ
   入力: ユーザークエリ
   ↓
   クエリをベクトル化
   ↓
   ナレッジベースを検索（コサイン類似度で計算）
   ↓
   関連ドキュメント TOP-K を取得

2️⃣ 増強フェーズ
   取得したドキュメント + クエリ
   ↓
   LLM への入力として結合

3️⃣ 生成フェーズ
   LLM が入力に基づいて回答を生成
```

**数学的流れ**:

$$\text{retrieval score}(q, d) = \cos(v_q, v_d)$$

$$\text{ranked docs} = \text{top-k}(\text{retrieval score})$$

$$\text{answer} = \text{LLM}(q + \text{ranked docs})$$

**プロジェクトでの実装**: [src/rag/agent.py](../../src/rag/agent.py)

### 2.3 言語モデル (Language Model)

**原理**: 単語の列が与えられたとき、次の単語の確率を計算

$$P(\text{単語}_t | \text{単語}_{1}, \text{単語}_{2}, \ldots, \text{単語}_{t-1})$$

```
例: "新しい治療法は"の次の単語の確率

P("効果" | "新しい治療法は") = 0.65     ← 最も確率が高い
P("高い" | "新しい治療法は") = 0.20
P("必要" | "新しい治療法は") = 0.10
その他                           = 0.05
                          ─────────────
                           合計 = 1.0
```

**テキスト生成の流れ**:
```
1. 最初の単語: P(最初の単語) から確率的に選択
2. 次の単語: 既存の単語列から P(次の単語 | 既存文) 計算
3. 繰り返す: 終了トークンまで繰り返す
4. 結果: 確率的に選択されたテキスト
```

### 2.4 ソフトマックス関数 (Softmax)

**役割**: 複数の値を確率分布に変換

$$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_j e^{x_j}}$$

```python
# 例: 3つの単語の候補
raw_scores = [2.0, 1.0, 0.5]
                ↓ softmax 適用
probabilities = [0.665, 0.244, 0.090]  # 合計 = 1.0
```

**プロジェクトでの使用**:
- ドメイン推定の確信度
- 単語生成確率の正規化

### 2.5 確率モデルの選択

**ドメイン推定の例**:

```
クエリ: "患者は新しい抗がん剤治療を受けている"

ドメイン候補:
  医療: P(医療|クエリ) = 0.92  ← 最も確度が高い
  法律: P(法律|クエリ) = 0.05
  技術: P(技術|クエリ) = 0.03

→ 医療ドメインを選択
```

**実装の数学**:

$$\text{domain} = \arg\max_{d} P(d|\text{query})$$

## 3️⃣ Level 3: 深掘り

### 3.1 トランスフォーマー・アーキテクチャ

**構成要素**:

```
入力テキスト
    ↓
[埋め込み層] → テキストを数値に変換
    ↓
[多頭注意層] ← 各単語間の関係を計算 ⭐ 核となる部分
    ↓
[フィードフォワード層] → 非線形変換
    ↓
[これらを繰り返す] × N層
    ↓
[出力層] → 次の単語の確率分布
```

### 3.2 注意機構 (Attention Mechanism)

**目的**: 入力文の中で「どの部分が重要か」を学習

**計算式**:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

ここで:
- $Q$ (Query): 「何を探しているか」
- $K$ (Key): 「各要素の特性」
- $V$ (Value): 「各要素の値」

**例**: "猫が魚を食べた" を解析

```
猫 が 魚 を 食べた

"食べた" という単語は：
  - "猫" と強く関連 (主語) → Attention 40%
  - "魚" と強く関連 (目的語) → Attention 50%
  - "が" や "を" は文法的 → Attention 10%

→ 文法構造と意味関係を自動学習
```

### 3.3 勾配降下法 (Gradient Descent)

**目的**: モデルの損失関数を最小化

```
損失関数 L(w) = (予測値 - 実際の値)²

勾配: ∇L = dL/dw  (損失がどの方向に増加するか)

更新式: w_{new} = w_{old} - α × ∇L

α: 学習率（ステップサイズ）
```

**視覚化**:

```
     損失
      ▲
      │     ●← 最適な重み
      │    /│\
      │   / │ \
      │  /  │  \
      │ /   │   \
      │┼────┼────┼─→ 重み(w)
      
勾配降下法は、この谷を下に向かって進む
```

**プロジェクトでの使用**:
- モデルの微調整
- プロンプト最適化での損失計算

### 3.4 バックプロパゲーション

**流れ**:

```
1. フォワードパス
   入力 x
     ↓
   [層1] → y₁
     ↓
   [層2] → y₂
     ↓
   [層3] → 出力
   
2. 損失計算
   Loss = f(出力, 正解)
   
3. バックワードパス（逆方向）
   出力層の勾配 ← 計算
     ↓
   層3の勾配 ← 計算（連鎖律を使用）
     ↓
   層2の勾配 ← 計算
     ↓
   層1の勾配 ← 計算
     ↓
   入力層の勾配 ← 計算
   
4. 重みの更新
   w ← w - α × 勾配
```

**連鎖律 (Chain Rule)**:

$$\frac{dL}{dw} = \frac{dL}{dy_3} \times \frac{dy_3}{dy_2} \times \frac{dy_2}{dy_1} \times \frac{dy_1}{dw}$$

### 3.5 エンコーダ・デコーダ・アーキテクチャ

**用途**: Seq2Seq モデル（翻訳、요약、質問応答など）

```
入力シーケンス        出力シーケンス
"新しい治療法は"  →  "エンコーダ" → "コンテキストベクトル"
                              ↓
                           "デコーダ" → "効果が高い"
```

**数学的表現**:

$$\text{context} = \text{encode}(x_1, x_2, \ldots, x_n)$$
$$\text{output} = \text{decode}(\text{context})$$

## 💻 実装演習

### 演習 1: ベクトルと類似度

```python
import numpy as np

# ベクトルの定義
doc1 = np.array([0.5, 0.3, 0.8])
doc2 = np.array([0.52, 0.28, 0.82])
query = np.array([0.1, 0.2, 0.3])

# コサイン類似度の実装
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 計算
print(f"doc1 と doc2 の類似度: {cosine_similarity(doc1, doc2):.3f}")  
# 出力: 0.999 (非常に似ている)

print(f"query と doc1 の類似度: {cosine_similarity(query, doc1):.3f}")  
# 出力: 0.857 (中程度)
```

### 演習 2: ソフトマックス関数

```python
import numpy as np

def softmax(x):
    exp_x = np.exp(x - np.max(x))  # 数値安定性の為
    return exp_x / exp_x.sum()

# ドメイン推定のスコア
scores = np.array([2.5, 1.2, 0.8])  # 医療, 法律, 技術

probs = softmax(scores)
print(f"確率: 医療={probs[0]:.3f}, 法律={probs[1]:.3f}, 技術={probs[2]:.3f}")
# 出力: 確率: 医療=0.665, 法律=0.244, 技術=0.090

# 最確度のドメイン選択
selected_domain = ['医療', '法律', '技術'][np.argmax(probs)]
print(f"選択ドメイン: {selected_domain}")
# 出力: 選択ドメイン: 医療
```

### 演習 3: 勾配降下法のシミュレーション

```python
import numpy as np
import matplotlib.pyplot as plt

# 損失関数: L(w) = (w-3)² (最小値は w=3 で 0)
def loss(w):
    return (w - 3) ** 2

def gradient(w):
    return 2 * (w - 3)  # dL/dw

# 勾配降下法
w = 0  # 初期値
learning_rate = 0.1
history = [w]

for _ in range(50):
    grad = gradient(w)
    w = w - learning_rate * grad
    history.append(w)

print(f"最終的な w: {w:.3f} (目標: 3.0)")
# 出力: 最終的な w: 2.999 (目標: 3.0)
```
