### L4: パフォーマンステスト（1-2時間）

**目的**: システムのパフォーマンスが本番環境基準を満たすことを確認

**パフォーマンス基準**:

| 処理種別 | 基準 | 許容ばらつき |
|---------|------|----------|
| クエリ前処理（単一） | <100ms | ±20ms |
| ドメイン推定 | <5ms | ±1ms |
| マルチドメイン統合 | <50ms | ±10ms |
| 全体レスポンス | <500ms | ±100ms |

**測定方法**:

```python
import time
from src.rag.query_preprocessor import Phase7QueryPreprocessor
from src.self_improvement.domain_knowledge import DomainKnowledgeManager

preprocessor = Phase7QueryPreprocessor()
manager = DomainKnowledgeManager()

# 1. クエリ前処理速度
query = "医療システムの技術的実装と法的考慮"
start = time.time()
result = preprocessor.preprocess(query)
elapsed = time.time() - start
print(f"クエリ前処理: {elapsed*1000:.2f}ms")

# 2. ドメイン推定速度（100回平均）
start = time.time()
for _ in range(100):
    manager.infer_domain_from_query(query)
elapsed = time.time() - start
avg_time = elapsed / 100
print(f"ドメイン推定平均: {avg_time*1000:.3f}ms")

# 3. ストレステスト（1000回連続実行）
start = time.time()
for _ in range(1000):
    preprocessor.preprocess(query)
elapsed = time.time() - start
print(f"ストレステスト（1000回）: {elapsed:.2f}秒")
```

**成功基準**:
- クエリ前処理: <100ms
- ドメイン推定: <5ms
- ストレステスト時エラーなし
- メモリリークなし（`ps` で確認）
