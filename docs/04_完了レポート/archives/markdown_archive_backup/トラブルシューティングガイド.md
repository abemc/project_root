## 🔍 よくある問題と解決方法

### ❌ 問題 1: ログイン失敗 (認証エラー)

**症状:**
```
Error: Authentication failed
Status: 401 Unauthorized
```

**原因（優先度順）:**
1. ❌ パスワードが間違っている
2. ❌ ユーザーがロックされている
3. ❌ MFA トークンが期限切れ
4. ❌ DB 接続不可
5. ❌ API ゲートウェイが応答しない

**診断:**

```bash
# 1. ユーザーアカウント状態確認
curl https://api.security-platform.com/v1/users/{user_id} \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq '.status'
# 期待値: "active"

# 2. MFA 設定確認
curl https://api.security-platform.com/v1/users/{user_id}/mfa \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq '.methods'

# 3. DB 接続テスト
psql -h db-primary.internal -c "SELECT COUNT(*) FROM users WHERE email='$USER_EMAIL';"

# 4. API ゲートウェイヘルスチェック
curl http://api-gateway:8080/health | jq '.status'

# 5. ログ確認
grep "Authentication failed" /var/log/platform/auth.log | tail -10
```

**解決手順:**

| 原因 | 解決方法 |
|------|--------|
| パスワード間違い | ユーザーにリセットメール送信 |
| ユーザーロック | `unlock_user.sh <user_id>` を実行 |
| MFA 期限切れ | 新しい MFA コード再発行 |
| DB 接続不可 | [DB 故障対応](#db-故障) 参照 |
| API 不応答 | API ポッド再起動 |

**実行例:**

```bash
# ユーザーのロック解除
./scripts/unlock_user.sh USR-20260417-001
# Output: ✓ User unlocked

# パスワードリセットメール送信
curl -X POST https://api.security-platform.com/v1/auth/password-reset \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com"}'
```

### ❌ 問題 2: MFA トークン生成エラー (TOTP)

**症状:**
```
Error: Invalid TOTP code
Message: Time skew detected
```

**原因:**
1. ❌ サーバー時刻がずれている
2. ❌ ユーザーデバイス時刻がずれている
3. ❌ TOTP シードが不正
4. ❌ ネットワークタイムアウト

**診断:**

```bash
# 1. サーバー時刻確認
date -u
# 期待値: 正確な UTC 時刻

# 2. NTP 同期確認
timedatectl
# 期待値: "System clock synchronized: yes"

# 3. TOTP シード確認
curl https://api.security-platform.com/v1/users/{user_id}/mfa/totp \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq '.seed_verified'
```

**解決手順:**

```bash
# サーバー時刻同期
sudo ntpdate pool.ntp.org

# または (systemd)
sudo systemctl restart systemd-timesyncd

# ユーザーに通知: デバイス時刻を確認するよう連絡
# → ユーザーデバイスの時刻設定を自動同期に設定

# TOTP 再登録
curl -X POST https://api.security-platform.com/v1/auth/mfa/totp/reset \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"user_id": "USR-20260417-001"}'
```

### ❌ 問題 4: SOC イベント処理遅延

**症状:**
```
Alert: SOC Queue Backlog
Queue Length: 15,000 events
Processing Latency: 2,500ms (目標: < 50ms)
```

**原因（優先度順）:**
1. ❌ イベント生成が多すぎる
2. ❌ イベント処理ワーカー数不足
3. ❌ メモリ不足
4. ❌ DB 書き込み遅延
5. ❌ ネットワーク問題

**診断:**

```bash
# 1. イベント生成レート確認
curl http://monitoring:9090/api/v1/query?query=soc_event_generation_rate | jq '.data'
# 期待値: < 10,000 events/sec

# 2. ワーカー確認
kubectl get deployment -n production soc-event-processor -o wide
# 期待値: Replicas: 10 / Ready: 10

# 3. メモリ使用率
kubectl top pod -n production -l app=soc-event-processor | awk '{sum+=$3} END {print sum}'

# 4. DB 遅延確認
psql -h db-primary.internal -c "SELECT extract(epoch from (now() - query_start)) FROM pg_stat_activity WHERE query LIKE '%INSERT INTO events%';"

# 5. キュー長確認
redis-cli -h cache.internal LLEN soc:event:queue
```

**解決手順:**

| 原因 | 解決方法 | 実施時間 |
|------|--------|--------|
| イベント生成多い | イベント フィルタ設定, 重複排除 | 10-15分 |
| ワーカー不足 | Pod レプリカ数増加 (スケーリング) | 5分 |
| メモリ不足 | ワーカーメモリ割り当て増加 | 15分 |
| DB 遅延 | クエリ最適化, インデックス確認 | 30分 |
| ネットワーク | ネットワーク構成確認 | 20分 |

**実行例:**

```bash
# Option A: イベント フィルタ設定 (短期対応)
curl -X PATCH https://api.security-platform.com/v1/security/soc/config \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{
    "event_filter": {
      "deduplicate": true,
      "min_severity": "medium"
    }
  }'

# Option B: ワーカー スケーリング (推奨)
kubectl scale deployment soc-event-processor -n production --replicas=20

# 確認
kubectl get pods -n production -l app=soc-event-processor | grep Running | wc -l
# 期待値: 20

# キュー復帰を監視
watch -n 1 'redis-cli -h cache.internal LLEN soc:event:queue'
# キューが減少したら → スケーリング成功
```

## ⚠️ エラーコード対応表

### 認証エラー (400-499)

| コード | メッセージ | 原因 | 対応 |
|--------|-----------|------|------|
| 400 | INVALID_INPUT | 入力値が不正 | 入力形式を確認 |
| 401 | UNAUTHORIZED | 認証失敗 | 認証情報を確認 |
| 403 | FORBIDDEN | 権限なし | 権限を確認・付与 |
| 405 | MFA_REQUIRED | MFA 認証が必要 | MFA トークン送信 |
| 429 | TOO_MANY_ATTEMPTS | ログイン試行多数 | 30分後に再試行 |

### サーバーエラー (500-599)

| コード | メッセージ | 原因 | 対応 |
|--------|-----------|------|------|
| 500 | INTERNAL_ERROR | サーバー内部エラー | ログを確認, 再実行 |
| 502 | BAD_GATEWAY | ゲートウェイエラー | API サーバー確認 |
| 503 | SERVICE_UNAVAILABLE | サービス不可 | 再開を待機 |
| 504 | GATEWAY_TIMEOUT | タイムアウト | 通信確認 |
