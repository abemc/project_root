# マークダウンフォーマット問題: 原因と解決策

**作成日**: 2026-04-11  
**対象ファイル**: PHASE7_IMPLEMENTATION_REPORT.md

---

## 🔍 問題の原因

### 1. **複数のマークダウン実装の異なる仕様**

| 実装 | 特性 | 課題 |
|-----|------|------|
| **markdown2** (変換用HTML/PDF) | CommonMark標準に準拠 | コードブロック内のモック-up を厳密に処理 |
| **VS Code プレビュー** | GitHub Flavored Markdown (GFM) | インデント行の解釈が異なる |
| **GitHub Markdown** | GFM実装 | リスト記法の検出が激しい |

### 2. **コードブロック内のインデント行の曖昧性**

**問題の例:**

```markdown
### 文脈理解の深さ

```
従来:
  - ユーザークエリ → ドメイン検出 → 知識検索 (3層)
```
```

**何が起こるか:**

- **markdown2**: コードブロック内なので、`-` をリスト記法として処理しない
- **VS Code**: インデント付き `- ` を「リスト継続」として解釈する可能性
- **GitHub**: GFMルールで独自に処理

### 3. **改行と行継続の処理差**

```markdown
Phase 7 後:
  - ユーザークエリ → 背景分析 → 明示的・隠れた意図検出 
    → ドメイン推定 → 複数ドメイン検索 → 知識統合 
    → 因果分析 → 不確実性評価 (8+ 層)
```

**問題:**

- 行の末尾に空白があると、マークダウンパーサーは行継続と判定する場合がある
- インデント量が異なると、「新しいリスト項目」と「リスト継続」の判定が曖昧になる

---

## ✅ 解決策: 標準化されたマークダウン記法

### 規則 1: コードブロック内は视觉的表現に統一

**❌ 悪い例:**
```markdown
```
従来:
  - 項目1
  - 項目2
```
```

**✅ 良い例:**
````markdown
```
従来:
  • 項目1
  • 項目2
```
````

（`-` の代わりに `•` を使用、またはコードブロック外にする）

### 規則 2: テキストの行継続は明示的に

**❌ 悪い例:**
```markdown
説明文が長い場合は
  行を分割
```

**✅ 良い例:**
```markdown
説明文が長い場合は、行を分割します。

複数行に渡る場合は：

- 項目1継続は缩进で示します
- 行間に空白を入れます
```

### 規則 3: コードブロック内の構造表現

**❌ 悪い例:**
```markdown
```
従来:
  - ユーザークエリ → ドメイン検出 → 知識検索
```
```

**✅ 良い例 (方法1: 矢印図表):**
```markdown
従来のフロー:
```
ユーザークエリ
    ↓
ドメイン検出
    ↓
知識検索
```
```

**✅ 良い例 (方法2: テキストベース):**
```markdown
従来: ユーザークエリ → ドメイン検出 → 知識検索 (3層)
```

### 規則 4: テーブル周辺の空白

**❌ 悪い例:**
```markdown
テーブル説明
| 列1 | 列2 |
```

**✅ 良い例:**
```markdown
テーブル説明

| 列1 | 列2 |
|-----|-----|
```

## 🔧 PHASE7_IMPLEMENTATION_REPORT.md の改善

### 適用する変更

| セクション | 変更内容 | 理由 |
|----------|--------|------|
| 文脈理解の深さ | コードブロックを ASCII アート図表に変更 | 視覚的で曖昧性がない |
| 隠れた意図認識 | `- ` の代わりに `•` を使用 | リスト記法との混同回避 |
| マルチドメイン知識統合 | 行継続にコードセルを使用 | インデント曖昧性の排除 |
| テーブル | 前後の空白を確認 | 確実なテーブル認識 |

## 📊 比較表: マークダウン実装ごとの変動

| 記法 | markdown2 | VS Code | GitHub | **推奨** |
|-----|---------|---------|--------|---------|
| コードブロック内 `- ` | × リスト化しない | ⚠️ 曖昧 | × リスト化しない | 避ける |
| 矢印 `→` | ✅ OK | ✅ OK | ✅ OK | 推奨 |
| 行末空白 + 改行 | ✅ 継続 | ✅ 継続 | ✅ 改行 | 避ける |
| テーブル前の空行 | ✅ 必須 | ✅ 推奨 | ✅ 推奨 | 必須 |
| インデント (2sp) | ✅ 可 | ✅ 可 | ✅ 可 | 推奨 |
| インデント (4sp) | ✅ コード | ✅ コード | ✅ コード | マークダウン本体用 |

## 🎯 最終推奨: 標準化戦略

### フェーズ1: 現在
- ✅ markdown2 (HTML/PDF変換): 完全互換性あり
- ⚠️ VS Code プレビュー: 微妙な表示差
- ✅ GitHub: 問題なし

### フェーズ2: 改善版 (推奨記法採用)
```markdown
# 全マークダウン実装で完璧に統一された表示を実現
```

### 具体的な変更

1. **コードブロック内のモック-up は避ける**
   - 代わりに: テキスト行頭に `→` 矢印を使用
   - または: ASCIIアート図表

2. **行継続は明示的な段落分けに変更**
   - 代わりに: 各行を独立した段落に

3. **インデント行は4スペースに統一**
   - またはテーブルやリスト記法に変更

4. **テーブル前後の空行を必ず確保**
