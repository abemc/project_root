## Claude MythosとRAGの比較（一般的観点）

| 観点             | Claude Mythos（Claude 3等）                | RAG（Retrieval-Augmented Generation）         |
|------------------|---------------------------------------------|-----------------------------------------------|
| アーキテクチャ   | 大規模言語モデル（LLM）単体                 | LLM＋外部知識検索（検索拡張生成）             |
| 知識の範囲       | 訓練時点までのコーパスに依存                | 外部データベース・ドキュメントも参照可能      |
| 回答の根拠       | モデル内部の知識に基づく                    | 検索結果（根拠文書）を明示的に参照・提示      |
| 拡張性           | モデル再学習が必要                          | データベースや検索対象を追加するだけで拡張可  |
| 精度・信頼性     | 一般知識・推論に強いが幻覚も発生しうる      | 根拠付き回答で信頼性向上（検索精度に依存）    |
| コスト           | API利用料・推論コスト                       | LLM＋検索分のコスト（構成次第で変動）         |
| ユースケース     | 汎用QA、要約、創造的生成、会話              | 法務・医療・社内QAなど根拠重視の業務用途      |
| 弱み・注意点     | 幻覚・根拠不明な回答のリスク                | 検索精度・データ整備が不十分だと誤答も発生    |

---

> Claude Mythosは高性能なLLMとして幅広い用途に対応しますが、RAGは「根拠付き・最新情報」の回答が求められる場面で特に有効です。

## RAGシステム強化・差別化計画書（2026年版）

### 1. 検索精度・データ整備の自動化
- 定期的な検索精度評価の自動実行（例：自動テスト＋レポート生成）
- コーパス・メタデータの品質管理ツール導入
- データ追加・更新時の自動インデックス再構築

### 2. フォールバック＆リカバリ自動化
- 検索失敗・インデックス破損時の自動フォールバック（例：従来LLM回答やキャッシュ利用）
- バックアップ・リストアの自動化、障害時の自動復旧フロー
- フォールバック発生時のユーザー通知・管理者アラート

### 3. 根拠提示のUI/UX強化
- 回答根拠文書のハイライト・出典リンク表示
- ユーザーによる根拠フィードバック機能（信頼性向上・改善サイクル）
- 根拠付き回答のテンプレート標準化

### 4. 知識の自動アップデート・鮮度維持
- 外部データソース・Webクローラ連携による自動データ収集
- 定期的なインデックス自動更新・古いデータのアーカイブ
- 最新情報取得時のユーザー通知

### 5. 幻覚（ハルシネーション）検知・抑制
- 根拠不明な生成を検知・警告するアルゴリズム導入
- 根拠必須モード（根拠がなければ回答しない）オプション
- 幻覚発生時の自動再検索・再生成フロー
