## コア機能の使用方法

### A) ドメイン自動推定

```python
from self_improvement.domain_knowledge import DomainKnowledgeManager

manager = DomainKnowledgeManager()

# 単一クエリの推定
query = "新しい治療法の臨床効果"
result = manager.infer_domain_from_query(query)
domain = result[0][0]        # 推定ドメイン
confidence = result[0][1]    # 信頼度

print(f"推定ドメイン: {domain}")
print(f"信頼度: {confidence:.2%}")
```

### B) バッチ処理

```python
queries = [
    "患者の医療情報と治療方針",
    "知的財産権の契約交渉",
    "デジタル化戦略の実装",
    "クラウドシステムのセキュリティ",
    "統計分析手法の応用"
]

for query in queries:
    result = manager.infer_domain_from_query(query)
    domain, confidence = result[0]
    print(f"{query}: {domain} ({confidence:.2%})")
```

### C) ドメイン情報の取得

```python
# 全ドメイン一覧
domains = manager.list_domains()

for domain in domains:
    print(f"📌 {domain.name}")
    print(f"   説明: {domain.description}")
    print(f"   キーワード数: {len(domain.key_concepts)}")
    print()
```

## トラブルシューティング

### 問題: ドメイン推定がうまくいかない

**症状**: `推定: general (信頼度: 50%)`

**原因**: クエリのキーワードが辞書に含まれていない

**対処**:
1. [domain_knowledge.py](src/self_improvement/domain_knowledge.py) を開く
2. 対応ドメインの `key_concepts` にキーワードを追加
3. システムを再起動

**例**:
```python
# business ドメインに新しいキーワードを追加
key_concepts = [...existing..., 'mynewtopic', '新しいキーワード']
```

### 問題: RAG統合が動作しない

**症状**: `warning: RAG connection failed`

**原因**: 接続設定が不正

**対処**:
1. [production_config.json](logs/production_config.json) を確認
2. RAG接続設定を修正
3. システムを再起動

### 問題: 応答が遅い

**症状**: 処理に数秒以上かかる

**原因**: システムリソース不足 または 大量クエリ

**対処**:
1. システムリソース使用率を確認
2. 必要に応じてバッチ処理に分割
3. キャッシュを有効化
