## 💡 学んだ教訓

### 1. マークダウン実装の違い
- CommonMark 基準: 厳密で一貫性がある
- GitHub Flavored Markdown: よりリッチな機能で拡張
- 統一性を保つには **保守的な記法** を使用すべき

### 2. 回避すべき記法
```markdown
❌ コードブロック内のリスト記法
❌ インデント + 行末空白での行継続
❌ テーブルの前後の空行不足
```

### 3. 推奨される記法
```markdown
✅ テーブルで複雑な構造を表現
✅ リストで階層構造を明確化
✅ 矢印 (→) で流れを表現
✅ 太字・斜体で強調
```
